## 🚀 快速开始

### 环境准备

```bash
# 安装 TypeScript
npm install -g typescript

# 检查版本
tsc --version

# 编译 TS 文件
tsc 基础类型基础.ts

# 运行编译后的 JS
node 基础类型基础.js
```

### VS Code 环境

1. **安装扩展**：
   - TypeScript Vue Plugin
   - ES7+ React/Redux/React-Native snippets

2. **查看类型**：将鼠标悬停在变量上（Ctrl+K Ctrl+I）

3. **快速修复**：Ctrl+. 查看建议

## 🎯 关键概念一览

| 概念 | 简单说法 | 例子 |
|------|--------|------|
| **类型注解** | 告诉变量是什么类型 | `let name: string = "张三"` |
| **接口** | 定义对象的形状 | `interface User { name: string }` |
| **泛型** | 可以用任何类型的"容器" | `Array<string>`, `Promise<User>` |
| **Union** | 可以是这个或那个 | `string \| number` |
| **readonly** | 定义后不能改 | `readonly id: number` |

## ⚠️ 常见问题

**Q：学 TS 有什么用？**  
A：类型检查能在编译阶段发现 bug，而不是运行时才发现。用 VS Code 时，自动完成会帮你想起 API。

**Q：会不会很难？**  
A：如果你会 JavaScript，TS 其实就是加了"类型标注"，不难。

**Q：一定要学 TS 吗？**  
A：对于公司项目，几乎必需。但理论上可以跳过，直接学 Koa，然后再补 TS。

**Q：Python 的类型注解和 TS 的类型是一样的吗？**  
A：概念相似，但 TS 的类型系统更强大。TS 类型会真正编译检查，Python 只是建议。
