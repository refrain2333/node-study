# 第二阶段：TypeScript 入门（开发标准）

> **学习周期**：3 天  
> **学习目标**：掌握 TS 类型系统，能够用 TS 开发简单项目  
> **前置要求**：已完成第一阶段 JavaScript 学习

---

## 📚 学习内容概览

### 1. 基础类型（1 天）
- **基本类型**：string, number, boolean, null, undefined
- **特殊类型**：any, void, never, unknown
- **字面量类型**：1, "hello", true
- **联合类型**：string | number
- **数组类型**：string[], Array<string>
- **元组类型**：[string, number]
- **枚举类型**：enum Direction { Up, Down }

### 2. 接口与类型别名（1 天）
- **Interface（接口）**：定义对象结构
- **Type（类型别名）**：可定义任何类型
- **区别和用途**：何时用 Interface，何时用 Type
- **继承和扩展**：extends 和 &
- **只读属性**：readonly
- **可选属性**：?

### 3. 泛型（1 天）
- **泛型基础**：<T> 的含义
- **泛型函数**：function fn<T>(arg: T): T
- **泛型接口**：interface List<T>
- **泛型类**：class Container<T>
- **泛型约束**：<T extends Something>
- **常见泛型**：Promise<T>, Array<T>

### 4. 类与装饰器（选学，但推荐）
- **类的基础**：constructor, 属性, 方法
- **访问修饰符**：public, private, protected
- **继承**：extends, super
- **抽象类**：abstract
- **装饰器**：@Decorator（Node.js 中间件常用）

### 5. 综合实战（1 天）
- **结合 JS 知识**：用 TS 重写 JS 项目
- **实际应用**：API 类型定义, 业务类型系统
- **常见模式**：CRUD 类型定义, 异步操作类型

---

## 🎯 学习策略

### ❌ 常见误区
- **不要过度设计类型**：复杂的类型不一定更好
- **不要滥用 any**：any 会失去 TS 的优势
- **不要背诵**：理解概念比记忆语法重要

### ✅ 正确做法
1. **边看文档边实践**：写 3 行代码 > 看 30 行说明
2. **从 Python 对比思维**：TS 的类型 ~ Python 的类型注解
3. **用 VS Code 的自动完成**：这是 TS 最大的优势
4. **逐步添加类型**：先写 JS，再加类型

## 💡 快速参考

### 最常用的类型定义（Koa 中间件例子）

```typescript
// 参数类型
interface Request {
  body: any
  query: Record<string, string>
}

// 函数类型
const handler = (ctx: Context): Promise<void> => {
  // 处理逻辑
}

// 泛型使用
async function fetchData<T>(url: string): Promise<T> {
  const response = await fetch(url)
  return response.json()
}
```

这些模式在第三阶段 Koa 开发中会反复出现！
